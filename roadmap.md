# План создания инструментов онбординга (Onboarding Toolset)

## 1. Концепция: "Onboarding Portal" (SPA + Backend)

**Цель:** Создать единую точку входа для новых сотрудников, автоматизировать рутину (чек-листы, доступы), геймифицировать процесс и собирать метрики для HR/Тимлидов.

**Стек:**

* **Frontend:** React (Vite), Tailwind CSS, React Query.
* **Backend:** Node.js (NestJS или Express), PostgreSQL/MongoDB.
* **Real-time:** WebSocket (Socket.io) — для уведомлений и статусов.
* **Integrations:** Slack/Telegram API, Jira API, GitLab API.

***

## 2. Архитектура и Функционал

### 2.1. Роли пользователей

1. **Newbie (Новичок):**
    * Видит свой прогресс-бар ("Квест первой недели").
    * Имеет список задач (чек-лист): "Настроить IDE", "Получить пропуск", "Сделать PR".
    * Видит контакты своего Ментора и Бадди.
    * Доступ к базе знаний (Wiki) и FAQ.
2. **Mentor/Buddy:**
    * Видит прогресс своих подопечных.
    * Получает уведомления, если новичок застрял.
    * Подтверждает выполнение ключевых этапов (например, "Code Review пройден").
3. **Admin/HR:**
    * Создает профили новичков.
    * Назначает менторов.
    * Смотрит аналитику: "Среднее время настройки окружения", "CSAT онбординга".

### 2.2. Основные модули системы

#### **Модуль 1: "Quest Map" (Интерактивный чек-лист)**

* **Frontend:** Список задач с группировкой по дням (Day 1, Week 1, Month 1).
* **Logic:**
    * Некоторые задачи закрываются вручную (галочка).
    * Некоторые — автоматически через API (например, скрипт проверил, что PR смерджен в GitLab → задача в портале закрылась сама).
* **Gamification:** Прогресс-бар, ачивки ("First Commit", "Setup Master").


#### **Модуль 2: "Team Radar" (Контакты и структура)**

* **Frontend:** Карточки команды с ролями.
* **Logic:** Показывает, кто за что отвечает (кто DevOps, кто Product Owner), чтобы новичок знал, кого пинговать.


#### **Модуль 3: "Automated Access" (Интеграции)**

* **Backend:** Скрипты на Node.js, которые дергают API сторонних сервисов.
    * *Пример:* При создании юзера → отправить инвайт в Slack, создать тикет в Jira на выдачу ноута.


#### **Модуль 4: "Feedback Loop" (Опросы)**

* **Logic:** В конце каждой недели всплывает мини-опрос (CSAT/eNPS).
* **Analytics:** Дашборд для HR с графиками удовлетворенности.

## 3. CLI-утилита "Onboarder" (как дополнение)

*Для автоматизации локальной настройки на машине разработчика.*

**Функции:**

* `onboarder doctor` — проверяет, установлен ли Node, Docker, Git, доступы к приватному npm.
* `onboarder setup` — одной командой клонирует репозитории, ставит зависимости, поднимает базу в Docker.
* `onboarder helpme` — отправляет алерт ментору в Slack с логами ошибки.

## 5. Технические челленджи (для портфолио)

* **Real-time state:** Синхронизация состояния чек-листа между новичком и ментором через WebSocket.
* **Microservices/Integrations:** Работа с внешними API (Jira, Slack, GitLab) — это ценный навык.
* **Security:** Правильное хранение токенов доступа к внешним системам.
* **DX (Developer Experience):** Написание удобной CLI-тулзы.
